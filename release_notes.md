# 🔥 v6.0.1 更新日誌

發佈日期：2026-04-07

本次 v6.0.1 版本全面重寫了首頁的異常偵測區塊，從原本單純的文字警示標籤，升級為互動式的詳細分析圖表卡片，幫助管理員與使用者能更快鎖定並了解問題根源。

## ✨ 新增功能與優化 (Features & Improvements)
- **視覺進度條與動態圖表 (Visual Data & Charts)**：新增獨立的異常展開卡片。卡片摘要即時顯示數量、佔比與上期變化，並加入「佔比視覺進度條 (Gauge Track)」。展開後直接以水平長條圖 (Horizontal Bar Chart) 顯示異常組成細節。
- **智慧洞察與說明 (Insights & Explanations)**：自動根據偵測結果生成上下文語境，並提供具體的管理建議，利用醒目顏色標註重點數值與時效。
- **相關案件內聯預覽**：異常卡片展開即可預覽最具代表性的前 5 筆相關案件（包含單號、客戶、機型與處理總時效 TAT），也可一鍵切換至完整明細彈窗。
- **響應式佈局支援**：大幅最佳化在平板與手機小螢幕上的閱讀體驗，自動折疊次要表格資訊與縮減按鈕空間。
- **底層效能與元件解耦**：將儀表板主視圖內建的龐大邏輯抽取至 `AnomalySection.jsx` 元件中，使 `DashboardView` 更乾淨好維護。

# 🔥 v5.9.1 更新日誌

發佈日期：2026-03-30

本次 v5.9.1 版本主要針對維修與保養之資料聚合邏輯進行大幅度的細項拆解與精確度提升，讓主管在審視報表時能更有效區分不同性質之派工狀態，並強化問題溯源功能。

## ✨ 新增功能與優化 (Features & Enhancements)
- **動態 SLA 目標天數對齊**：修正明細彈窗內硬編碼的 SLA 逾期計算方式，全面改為依據 `calculations.js` 內配置的動態目標天數進行判定。
- **全動態廠牌與設備類型分類**：自動判斷機型字串輸出分類標籤（如 Philips、ResMed、萊鎂 等），同步支援 CPAP、BiPAP 等設備細分類，UI 完全解耦。
- **保固與保養資料深度解耦**：將「定期保養」從設備維修中獨立分流，並進一步細分出「居家保養」與「醫院保養」。
- **全局板塊「查看明細」功能**：於「四大業務板塊分析」中新增防呆機制的「查看明細」按鈕，直接點擊查閱該板塊完整工單紀錄清單。
- **保固差異與「誤差名單」**：將「簡易檢測」納入真實保固維修，新增「誤差名單」對比機制，一鍵揭露潛在的標籤錯植案件。

## 🐛 錯誤修正 (Bug Fixes)
- 修正點擊「四大板塊」數值時，因事件冒泡（Event Bubbling）導致的畫面熱更新失效及 UI 預設阻擋行為。
- 補齊 `getCategory` 的系統匯入指令，避免操作與過濾分類時發生 JS ReferenceError 及白屏現象。

# 🔥 v5.9.0 更新日誌

發佈日期：2026-03-21

本次 v5.9.0 版本為系統帶來了巨大的底層重構與體驗提升，包含了 SLA 算力的優化以及儀表板 UI 的全面升級。

## ✨ 新增與優化功能 (Feature & Enhancements)

### 1. 商業邏輯配置深度解耦 (Business Logic Decoupling)
- **獨立 `KpiConfig` 介面**：將所有硬編碼 (Hardcoded) 的 KPI 權重、SLA 得分階梯、服務類型點數對應表全數抽離至集中管理物件，未來調整營運政策（如修改超急件點數或召回率容忍度）不再需要更動核心迴圈程式碼，大幅降低維護風險。
- **支援台灣相容日曆**：全面升級 `getWorkingDays` 函數，完整支援行政院人事行政總處的「國定假日 (Holidays)」排除與「補班日 (Make-up Days)」納入計算，結合 $O(1)$ 查詢速度優化，確保淨完修天數 (Effective TAT) 的 SLA 判定 100% 精準。
- **Gini 係數演算法優化**：針對大規模資料集改善 Time Complexity。

### 2. UI/UX 與防呆機制升級 (UI Refactoring & UX)
- **戰略指標數據卡片 (KpiSummaryCard) 重構**：全專案引入 Tailwind CSS (v4) 機制重寫指標頂層卡片。
- **防禦性介面設計 (Defensive Design)**：新增精準的骨架屏載入動畫 (Skeleton State) 避免讀取卡頓，並實作 Empty State (空資料) 與 Error State (異常) 的攔截保護，再也不會因為單一 `NaN` 或髒資料導致畫面崩壞。
- **自適應排版與動效**：導入柔性的卡片 Hover 浮動動畫、漸層隱晦光暈，並精準控制 Flexbox 與 CSS Grid 確保所有螢幕尺寸的完美相容。修正深淺色模式 (Dark/Light mode) 的主題樣式相容性。

### 3. 開發者文件全面補齊
- 新增 `DEVELOPER_GUIDE.md` 完整記錄全端資料流與元件分層解析，提供未來交接與擴充新圖表的標準 SOP。
